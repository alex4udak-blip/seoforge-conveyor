# SEOForge — полный конвейер (оркестрация). Модель потока Алекса 01.06

Сценарий: «разведчик идёт в Tier-1 (10 стран) → вывод по ключам/версткам → Dynadot ищет домен+цена →
задание на генерацию+SEO → копирайт-проверка → AI-SEO агент (Reddit/entity) → регистрация → деплой → трекинг».

## ПОТОК (агенты передают данные по цепочке)

```
┌─ 1. РАЗВЕДЧИК (RECON) ──────────────────────────────── ✅ ГОТОВ
│   вход: список ключей × 10 гео Tier-1
│   делает: топ выдачи, официал/сеошник, возраст доменов, footprint,
│           динамика (мясорубка/статика), семантика-куст (related/PAA), botview-клоака
│   ВЫВОД: "ключ X: N трафика, конкуренция=сеошники на трастах 17-25 лет,
│           НО есть свежий 5мес → заходимо. Типы сайтов в топе: WP-toplist.
│           Щель: long-tail [...]. Делать: обзорник+AI-слой, 2-3 варианта вёрстки"
│   статус: работает на проде (/recon). Не хватает: скоринг СИЛЫ траста (DR/бэклинки) для решения бить/нет
└──────────────┬──────────────────────────────────────
               ▼ передаёт: {ключ, гео, тип_сайта, семантика, конкуренты}
┌─ 2. ДОМЕНЩИК (Dynadot) ─────────────────────────────── ✅ ГОТОВ (модуль есть)
│   вход: бренд-идея под ключ/гео
│   делает: domain_check (доступность+цена по зонам) + domain_score (траст по Wayback)
│           + marketplace (262 готовых гембл-трастов с бэклинками $26-49)
│   ВЫВОД: "домен luckyaviator-casino.click $1.50/год (расходник Tier-3)
│           ИЛИ траст-дроп из базы 10bet-app.net $30/581 бэклинков (Tier-1 заход)"
│   статус: tools/ в SEO-Scanner-Pro работают. Не хватает: подключить в оркестратор + авто-выбор домена
└──────────────┬──────────────────────────────────────
               ▼ передаёт: {домен, цена, тип(расходник/траст)}
┌─ 3. ГЕНЕРАТОР + SEO (агентный) ─────────────────────── 🔶 ЕСТЬ v1 (шаблонный), НУЖЕН агентный
│   вход: {ключ, гео, конкуренты из RECON, семантика-куст, домен}
│   ДОЛЖЕН (модель Димы): агент идёт в выдачу (RECON-данные) → изучает структуру топа →
│           верстает УНИКАЛЬНЫЙ сайт под этот ключ (не один шаблон!), нужные типы страниц
│           (toplist / brand-review / vs-long-tail / FAQ), гео-слой (валюта/платёжки)
│   сейчас: assembler3 = 1 каркас + вариации (footprint-риск, как старый SEO-site-)
│   статус: v1 на проде (/generate). ФАЗА B = переписать на агентную генерацию
└──────────────┬──────────────────────────────────────
               ▼ передаёт: {сайт (страницы), мета, schema}
┌─ 4. КОПИРАЙТ-ПРОВЕРКА (human-score) ────────────────── ❌ НЕТ
│   делает: GPT-Zero/детектор → human 80%+ (иначе бан за спам), уникальность,
│           перефраз если ниже порога. Оценка качества текста.
│   статус: не построен. Нужен: модуль проверки + перегенерация слабых текстов
└──────────────┬──────────────────────────────────────
               ▼
┌─ 5. AI-SEO АГЕНТ (параллельно) ─────────────────────── ❌ НЕТ (частично vision)
│   делает: AI-выдача оптимизация (inverted pyramid, FAQPage, entity/Wikidata),
│           внешние сигналы — посты Reddit/Quora со ссылкой, соц-профили,
│           Trustpilot отзывы из ЦЕЛЕВОГО гео, регистрация где надо
│   статус: vision-аудит есть (оценка), сам AI-SEO не автоматизирован
└──────────────┬──────────────────────────────────────
               ▼
┌─ 6. ИНДЕКСАЦИЯ ─────────────────────────────────────── ❌ НЕТ
│   делает: GSC/Bing API (квота ~300/акк), свой индексатор, sitemap-пинг
│   статус: не построен
└──────────────┬──────────────────────────────────────
               ▼
┌─ 7. КЛОАКА (CTS) ───────────────────────────────────── 🔶 ЕСТЬ отдельно (cloaking-project)
│   делает: серверная клоака device-fingerprint, white/black по гео
│   статус: код есть в cloaking-project, не интегрирован в конвейер
└──────────────┬──────────────────────────────────────
               ▼
┌─ 8. ДЕПЛОЙ В СЕТЬ ──────────────────────────────────── ✅ ГОТОВ
│   делает: сайт → Saturn app → свой домен/поддомен (Saturn API токеном)
│   статус: работает (генерация на проде + volume), деплой по API освоен
└──────────────┬──────────────────────────────────────
               ▼
┌─ 9. ТРЕКИНГ (Keitaro) + замыкание ──────────────────── ❌ НЕТ интеграции
│   делает: депы/EPC/постбэки из Keitaro → какой сайт/гео/оффер даёт деп →
│           скоринг назад в РАЗВЕДЧИК/ГЕНЕРАТОР (что повторять)
│   статус: Keitaro API есть в памяти, не подключён к конвейеру
└──────────────────────────────────────────────────────
```

## ИТОГ готовности по твоему сценарию
| Звено | Статус |
|---|---|
| 1. Разведчик ключей/топов/вёрсток | ✅ работает (нужен скоринг силы траста) |
| 2. Dynadot домен+цена | ✅ работает (нужно подключить в поток) |
| 3. Агентный генератор+SEO | 🔶 v1 шаблонный → нужен агентный (фаза B) |
| 4. Копирайт-проверка human | ❌ |
| 5. AI-SEO агент (Reddit/entity) | ❌ (vision-оценка есть) |
| 6. Индексация GSC/Bing | ❌ |
| 7. Клоака | 🔶 есть отдельно |
| 8. Деплой | ✅ |
| 9. Keitaro замыкание | ❌ |

ГОТОВО 3 из 9 полностью + 2 частично. Ядро (разведка+домены+деплой) есть.
Главный недостающий мост: ОРКЕСТРАТОР, что гонит данные по цепочке 1→9 автоматически.
